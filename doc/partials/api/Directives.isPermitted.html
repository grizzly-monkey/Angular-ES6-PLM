isPermitted
service in module Directives

Description

Hides the element if user does not have permission to see, or if the permission hasn't been supplied.

-workspaceId: This is an optional attribute that can be used with this directive to apply permission for the workspace, the id of which is provided as a value of this attribute. This is very useful to control visibility of the element based on other workspaces. Possible usage in global quick create.

-applyLock: This is an optional attribute that can be used with this directive to take workspace locking in to consideration while evaluating the visibility of the element. Possible usage for Add/Edit/Delete buttons on Item Details / Managed View / Grid and Relationship view.

-applyRevisionOverrideLock: This is an optional attribute that can be used with this directive to take revision override locking in to consideration while evaluating the visibility of the element. Note: This attribute is only applicable for revision controlled workspace item, for the rest this will be ignored. Possible usage for Add/Edit/Delete buttons on Item Details / Managed View / Grid and Relationship view.

Dependencies

  • Requires Services/PermissionService

Usage

as attribute
<ANY is-permitted>
   ...
</ANY>

Methods

  • updateVisibility(isVisible)

    A helper method to remove the element based on whether user has permissions. It is safe to remove since permissions will not change mid-view. When the element is removed, we can then use CSS selectors to style the primary button on the command bar.

    Parameters
    ParamTypeDetails
    isVisibleBoolean

    True to display the element.

Properties

  • applyLock

    Reflects a value of an OPTIONAL attribute that can be used with this directive to add workspace locking check while evaluating workspace id.

  • applyRevisionOverrideLock

    Reflects a value of an OPTIONAL attribute that can be used with this directive to add workspace revision control locking check while evaluating workspace id.
