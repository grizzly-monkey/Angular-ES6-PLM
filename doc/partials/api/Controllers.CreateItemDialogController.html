CreateItemDialogController
service in module Controllers

Description

The controller for create item dialog. This controller handles the dialog commands such as saving and closing.

Dependencies

Methods

  • addItem()

    Add the current item to the managed items tab of the newly created item

  • canSaveManagedItem()

    To check if managed item form data is in dirty state.

    Returns
    Boolean

    True, if managed item form data is ready for save.

  • clearFields()

    Reset fields to their default values

  • close()

    Closes create item dialog

  • createManagedItemFormObj()

    Create managed item form object by fetching required information related to managed item.

    Returns
    Object

    A promise for managed item form object.

  • getAvailableTransitions(itemId, itemRecord, availTransitLink)

    get available transition for a given managed item state.

    Parameters
    ParamTypeDetails
    itemIdString

    The id of the item.

    itemRecordObject

    The details of the item.

    availTransitLinkString

    Link to fetch information of transitions.

    Returns
    Object

    A promise that will be resolved when transitions are fetched.

  • getFields()

    Returns a flat array of all the fields in the currently selected workspace

    Returns
    Array

    List of all the fields

  • getManagedItemCustomFields()

    Create custom fields from given managedItem Models$AffectedItem and meta Models#AffectedItemMeta objects.

    Returns
    Array

    List of custom field information of managed item instance.

  • getManagedItemFields()

    Create form fields from given managedItem Models$AffectedItem object.

    Returns
    Object

    Field information of managed item instance.

  • isFormFilled()

    Checks if all fields in the form have been filled

    Returns
    Boolean

    True, if all fields in the form have been filled

  • isSaveDisabled()

    Checks if the save button should be disabled, depending on whether we're on the create or managed screen.

    Returns
    Boolean

    True, if the save button should be disabled

  • manage()

    Saves the affected items fields of the item. The method also handles save conflicts in-case of concurrent modifications.

    Returns
    Object

    A promise that will be resolved for successful save or rejected for unsuccessful attempt.

  • manageAndClose()

    Manages the item and closes the dialog

  • manageAndView()

    Manages the effectivity and lifecycle of the item and navigates to the affected items tab of the newly created item

  • onSaveChange()

    Upon selecting save type, do the respective save

  • save()

    Saves a newly created item

  • saveAndClose()

    Saves the new item and closes the dialog

  • saveAndCopy()

  • saveAndManage()

    Saves the new item and moves to the next screen of dialog

  • saveAndNew()

    Saves the new item and clears the form field so that a new item can be created

  • saveAndView()

    Saves the new item and navigates to the newly created item

  • updateManagedItemObject()

    Update managed item object managedItemObj with the form data managedItemFormObj.

Properties

  • createType

    String to determine the type of create

  • createdItemObj

    The newly create item object (used for in-contextual create, in the manage state)

  • currentItem

    The current item object

  • currentWorkspace

    The current workspace object

  • dialogElement

    The current dialog element

  • filterToApply

    The type of filter to apply on the fields

  • formFields

    An array consisting of the form fields

  • isContextualCreate

    True, if the dialog to be triggered is for in-contextual create and add to managed items tab

  • isManagedView

    Boolean to determine whether manage screen is displayed

  • isQuickCreate

    True, if the dialog to be triggered is for quick create

  • isSavingData

    Set to true when saving data is underway, as to block form elements from being edited while saving

  • managedItemFormObj

    An object that contains details of the managed item fields.

  • managedItemObj

    An instance of managed item model.

  • managedViewHeading

    The heading for the managed view

  • selectedSaveType

    The type of saving selected

  • selectedWorkspace

    The pre-selected workspace, aka the only permitted workspace in which the user can add item to for quick create

  • unsupportedFields

    An object updated by Directives.createItem to indicate if there are any unsupported fields

  • viewPermission

    The view items permission for the user

  • workspacesList

    List of permitted workspaces that a user can add item to
