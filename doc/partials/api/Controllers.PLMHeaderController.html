PLMHeaderController
service in module Controllers

Description

This controller is the application header, present in almost all pages.

Dependencies

Methods

  • goToClassicDelegations()

    Navigates to ClassicPLM delegations

  • goToClassicProfile()

    Navigates to ClassicPLM user profile

  • logOut()

    Logs out from the application

  • setUpPageHeader()

    private Sets up the header (e.g. title set in the app.js if it's available in the route params, plus buttons and other dropdowns)

  • switchToClassic()

    Navigates to ClassicPLM after saving the interface style value as Classic PLM

Properties

  • currentUserListenerId

    private Stores the listener for the currently logged in user

  • currentWorkspace

    The current workspace object, if in workspace context

  • headerText

    The title of the page (label that will go at the top)

  • isAddItemPage

    private Stores the listener for the item data

  • isLogged

    Flag for whether user is currently logged

  • itemListenerId

    private Stores the listener for the item data

  • searchQuery

    Holds the value in the query param of the url for the search PLM directive

  • userAvatar

    Object containing local avatars - to be overwritten as soon as user info comes back

  • userDisplayName

    The logged in user name - to be overwritten as soon as user info comes back

  • userEmail

    The logged in user email - to be overwritten as soon as user info comes back

  • workspaceErrorListenerId

    The reference of the workspaces errors listener

  • workspaceId

    The workspace id

  • workspaceTypeId

    private Stores the current workspace type id that is loaded
