ViewMilestonesController
service in module Controllers.workspaceItem

Description

This controller corresponds to /workspaceItems/{workspaceId}/{itemId}/milestones.

Dependencies

Methods

  • isViewState()

    Determine whether we're in view mode

    Returns
    Boolean

    true if we are in view mode

  • loadMilestones()

    Render Milestones Page depending on the mode (view, edit, create)

    Returns
    Boolean

    true if we are in view mode

  • parseMilestonesData()

    Initialize the table data.

  • parseWorkflowStates()

    Initialize the workflow states for dropdown in edit mode.

  • setupTableColumns()

    Initialize the table columns, dependent on the edit mode.

  • triggerCancel()

    Triggered by clicking cancel button.

    NOTE: name has to be triggerCancel.

  • triggerEdit()

    Triggered by clicking edit button - goes to the EDIT state.

  • triggerSave()

    Triggered by clicking save button.

    NOTE: name has to be triggerSave.

Properties

  • MilestoneObj

    The reference to milestones items data model object.

  • StatesObj

    The reference to workflow states object.

  • addPermission

    The add milestones items permission for the user.

  • currentUserListenerId

    private Listener for current user.

  • dateFormat

    The format of the date string the user has chosen.

  • edit

    The state of the editable fields shown in the TDW (this has to be an object so that any changes in it get automatically reflected in the fieldData)

  • editPermission

    The edit milestones items permission for the user.

  • itemId

    The item id of the current selected item.

  • itemListenerId

    Listener for item object. Determines the workspace type to initialise the revisioning columns, if needed. And gets the affected item custom columns meta data.

  • milestoneListenerId

    Listener for milestones object.

  • milestoneStatesListenerId

    Listener for workflow states object.

  • moment

    Moment is required to convert date format not supported by IE to a supported format

  • stateChangeListenerId

    Listener for state changes. When the state changes, update the edit object to reflect that change. Also set the table headers again because some fields are hidden in edit mode.

  • tableColumns

    Stores the details of the table's fields.

  • tableData

    Stores table data of the affected items.

  • viewPermission

    The view milestones items permission for the user.

  • workspaceId

    The workspace id of the current selected item.
