BomTableRow
service in module Models

Description

Class defining the interface for bom table rows

Methods

  • addRowErrors(errors)

    Updates the row with errors

    Parameters
    ParamTypeDetails
    errorsArray

    An array of row level errors

  • clearRowErrors()

    Removes any row level errors

  • constructor()

    initializes the Bom Table Row properties

  • getFieldWithSemantics(semantics)

    Returns the field that has the given semantics Currently we assume that only one field will exist with the given semantics. This will need to be changed if we get multiple fields with the same semantics

    Parameters
    ParamTypeDetails
    semanticsBomUIFieldSemantics

    The semantics of the field

    Returns
    BomFIeldData

    the data corresponding to the revision field, or null if the field is not found

  • getFields()

    Returns an array of the row's fields NOTE: This is relatively slow, because we call Object.keys to enumerate the display properties

    Returns
    Array

    An array of BomFieldData instances in the row

  • hasErrors()

    Determines if the row contains errors or not

    Returns
    Boolean

    True if there are errors associated with the row or its fields

  • hasLoadedRevisions()

    Returns true if the row has a revision field that has loaded options Revision.options may be undefined (if it were never set) or it may be null if it were initialized and not set during the construction of the field Otherwise, it will be an array, which is coerced to true

    Returns
    Boolean

    true if there is a revision field and it has some revisions associated with it

  • processFields()

    Updates the row's properties by analyzing its fields Currently used to keep a cached list of fields with errors, so that we don't need to reenumerate every time

  • updateRevisionField(versions)

    Adds the needed revision as options for the revisions column (if it is available)

    Parameters
    ParamTypeDetails
    versionsArray

    A list of revisions

Properties

  • $$treeLevel

    The depth of the row

  • clickHandler

    handler function triggered on click

  • depth

    The depth of the row

  • displayProperties

    Properties we are going to display in the table NOTE: Although UI-Grid purports to support arbitrary angular expressions for cell lookup, It doesn't appear to like anything but plain object property accessing, So display properties have to be stored as keys of an object, instead of entries in a Map

  • edgeId

    Id of the edge that was used to populate this row

  • hasFetchedChildren

    boolean to determine if the row's children have been fetched

  • hasFieldErrors

    Cache of fields that currently have errors associated with them Cached because hasErrors is called every digest cycle for every visible row, and traversing all the fields is relatively expensive.

  • isCheckable

    boolean to determine if the row should be checkable

  • isChecked

    boolean to determine if the row is checked

  • isCollapsed

    boolean to determine if the row is collapsed or not

  • isExpandable

    boolean to determine if the row is expandable

  • isNewlyAdded

    boolean to determine if the row was newly added and does not yet have a bomId on the server

  • item

    item information for this row

  • itemNumber

    Item number in relation to siblings

  • nodeId

    Id of the node that was used to populate this row

  • path

    The array of edges that leads to the row, starting with the root as null

  • rowErrors

    list of errors associated with the row

  • rowId

    id that identifes this row in a table
