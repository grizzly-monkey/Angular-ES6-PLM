LocalUserStorageService
service in module Services

Description

A utility service that stores non-secure data specific to the user locally

Methods

  • canUseLocalStorage()

    Becase browsers have weird ways fo dealing with localStorage, we need to check if we can use it Source: https://developer.mozilla.org/en-US/docs/Web/API/Web_Storage_API/Using_the_Web_Storage_API Does not guarantee that there will be enough room in localStorage for all the data

    Returns
    Boolean

    True if local storage can be used

  • constructor()

    The class constructor Immediately loads the current user

  • get(key)

    Get the value stored for the given key Assumes that local storage is available

    Parameters
    ParamTypeDetails
    keyString

    The key corresponding to the requested data

    Returns
    Object

    The requested object

  • getUserDataStorage()

    Gets the object that stores the user data If no object exists, creates one, stores it, and returns it Assumes that local storage is available

    Returns
    Object

    An object corresponding to the current user

  • loadUser()

    Loads the current user

  • set(key, value)

    Stores the value with the provided key in the user's local storage Assumes that local storage is available

    Parameters
    ParamTypeDetails
    keyString

    The key to use (must be usable as a key of an object)

    valueObject

    An object that will be stored The object must be stringifyable by JSON

Properties

  • userKey

    The key used to store the current user
