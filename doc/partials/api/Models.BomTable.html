Properties

  • changeTracker

    Tracker to track edits, adds, and removes while in edit mode

  • columns

    Columns Collection to display in the table

  • currentAddOrder

    The order in which rows gets added to the table. We use this order in the bomRowCompare method to keep the newly added row always at the end of the table.

  • edgesToFirstLevelItemNumbers

    This will contain all the first level item numbers .At any given state of the bom this will contain the top level item numbers that are currenly shown in the table. This will significantly reduce the number of elements we have to visit in order to update the current maximum item number. Also This will help us not re-order the bom table when an edit happens to the item number of an existing row.

  • edgesToItemNumbers

    Map of edges to their item numbers (as numbers), so we can sort by them We need this because although rows store thier itemNumber, they don't store their parents item numbers, and we can't order two paths (which may be significantly different) without that

  • maxFirstLevelItemNumber

    The maximum item number of the first level rows of Table

  • rowIdColumn

    Rows Collection to display in the table

  • rowIdColumn

    Set tracking the rows we have created already as a set of paths represented as strings (since we can't key on arrays)
